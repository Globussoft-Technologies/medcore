# Prisma Model Proposal: `AdherenceDoseLog`

> **Status:** Proposal — not yet merged into `packages/db/prisma/schema.prisma`.
> The schema-merger agent owns the actual schema edit. Until merged, the
> route handler casts through `(prisma as any).adherenceDoseLog` and carries
> a `// TODO(cast): remove after prisma generate` comment.

This model persists per-dose events for the medication-adherence feature. The
mobile app writes one row each time a patient marks a reminder chip as taken
(or skipped), and reads the last N days to hydrate the UI on focus.

## Model

```prisma
model AdherenceDoseLog {
  id              String   @id @default(cuid())
  scheduleId      String
  patientId       String
  medicationName  String
  scheduledAt     DateTime
  takenAt         DateTime?
  skipped         Boolean  @default(false)
  note            String?
  createdAt       DateTime @default(now())

  schedule        AdherenceSchedule @relation(fields: [scheduleId], references: [id], onDelete: Cascade)

  @@index([scheduleId])
  @@index([patientId])
  @@index([patientId, scheduledAt])
  @@map("adherence_dose_logs")
}
```

## Field rationale

| Field            | Notes                                                                                                |
| ---------------- | ---------------------------------------------------------------------------------------------------- |
| `id`             | `cuid()` — keeps write path cheap and collision-safe for client-generated correlation if ever needed |
| `scheduleId`     | FK into `AdherenceSchedule`. Cascade on delete so orphan logs cannot survive an unenroll             |
| `patientId`      | Denormalised for fast per-patient time-series queries without joining the schedule                   |
| `medicationName` | Denormalised from `AdherenceSchedule.medications` JSON — that array is mutable, this row is a fact   |
| `scheduledAt`    | The reminder slot the dose was logged against (UTC ISO)                                              |
| `takenAt`        | Nullable — may be `null` if `skipped=true` or if the patient marked it late without a concrete time  |
| `skipped`        | Explicit skip signal for adherence-score calculations                                                |
| `note`           | Optional free-text ("felt nauseous", etc.) — capped client-side, no server-side limit today          |
| `createdAt`      | Server-side timestamp for audit, distinct from `takenAt` which is patient-reported                   |

## Indexes

- `@@index([scheduleId])` — list doses for a schedule (used by `GET /:scheduleId/doses`)
- `@@index([patientId])` — list all doses for a patient (future analytics)
- `@@index([patientId, scheduledAt])` — composite to accelerate the per-patient
  time-window scan used for the mobile hydrate-on-focus call and future
  adherence-rate dashboards
