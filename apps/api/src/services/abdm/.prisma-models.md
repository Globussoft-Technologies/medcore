# ABDM Prisma Models (proposed)

These models must be added to `packages/db/prisma/schema.prisma` before running
the ABDM routes in production. They are kept here separately because the task
rules forbid editing `schema.prisma` directly.

The ABDM service files in this folder access these models via
`(prisma as any).<modelName>` so the code compiles against the current
schema — once these are migrated in, the `as any` casts can be dropped.

---

## `AbhaLink`

Binds a MedCore `Patient` to an ABHA Health Account. A patient may, in
theory, have multiple historical links (for audit), but at most one should
be in state `LINKED` at a time — enforce this via a partial unique index in
a follow-up migration if needed.

```prisma
model AbhaLink {
  id            String    @id @default(uuid())
  patientId     String
  abhaAddress   String    // e.g. "sumit@sbx"
  abhaNumber    String?   // 14-digit: "12-3456-7890-1234"
  status        AbhaLinkStatus @default(PENDING)
  requestId     String?   @unique              // correlation id for async Gateway callback
  linkedAt      DateTime?
  revokedAt     DateTime?
  failureReason String?
  createdAt     DateTime  @default(now())
  updatedAt     DateTime  @updatedAt

  patient       Patient   @relation(fields: [patientId], references: [id], onDelete: Cascade)

  @@index([patientId])
  @@index([abhaAddress])
  @@index([status])
  @@map("abha_links")
}

enum AbhaLinkStatus {
  PENDING     // Gateway link/init dispatched, awaiting callback
  VERIFIED    // Existence check passed, not yet fully linked
  LINKED      // Active binding
  REVOKED     // Patient or clinician explicitly de-linked
  FAILED      // Gateway returned a terminal failure
}
```

## `ConsentArtefact`

Stores the full ABDM consent artefact — both our outbound request and the
signed artefact returned by the Consent Manager. `artefact` is an opaque
JSON blob (the signed CM payload) so we preserve forward compatibility
with future ABDM spec versions.

```prisma
model ConsentArtefact {
  id            String    @id @default(uuid())   // same as consentRequestId
  patientId     String
  hiuId         String                           // our HIU id issued by ABDM
  purpose       String                           // CAREMGT | BTG | PATRQT | ...
  status        ConsentStatus @default(REQUESTED)
  artefact      Json                             // signed CM artefact
  expiresAt     DateTime
  grantedAt     DateTime?
  revokedAt     DateTime?
  createdAt     DateTime  @default(now())
  updatedAt     DateTime  @updatedAt

  patient       Patient   @relation(fields: [patientId], references: [id], onDelete: Cascade)

  @@index([patientId])
  @@index([status])
  @@index([expiresAt])
  @@map("consent_artefacts")
}

enum ConsentStatus {
  REQUESTED    // CM notified, awaiting patient decision
  GRANTED      // Patient approved; artefact signed and ready to use
  DENIED       // Patient rejected
  REVOKED      // Previously granted, then revoked
  EXPIRED      // Past expiresAt
}
```

## `CareContext`

Tracks which patient care-contexts we have advertised to the Gateway so
the ABHA app can discover them. `careContextRef` is the opaque MedCore-side
identifier sent to ABDM (e.g. `scribe:<sessionId>`); it must be unique so
we can idempotently re-push the same context.

```prisma
model CareContext {
  id              String   @id @default(uuid())
  patientId       String
  abhaAddress     String
  careContextRef  String   @unique              // "scribe:<sessionId>", "discharge:<admissionId>", ...
  type            CareContextType
  lastPushedAt    DateTime?
  createdAt       DateTime @default(now())

  patient         Patient  @relation(fields: [patientId], references: [id], onDelete: Cascade)

  @@index([patientId])
  @@index([abhaAddress])
  @@map("care_contexts")
}

enum CareContextType {
  OPConsultation
  DischargeSummary
  DiagnosticReport
}
```

## Migration notes

1. Add the three `enum`s and three `model`s above to
   `packages/db/prisma/schema.prisma`.
2. Add the reverse relations on the `Patient` model:
   ```prisma
   abhaLinks         AbhaLink[]
   consentArtefacts  ConsentArtefact[]
   careContexts      CareContext[]
   ```
3. Run `npx prisma migrate dev --name add_abdm_tables` in `packages/db`.
4. After the migration lands, remove every `(prisma as any)` cast in
   `apps/api/src/services/abdm/*.ts` — there are roughly 9 of them.
