# Proposed Prisma models for the RAG ingest / chart-search pipeline

These models are **not yet added** to `packages/db/prisma/schema.prisma`. The
scope for this feature was to keep schema changes out of scope; the ingest
pipeline works today by relying on the existing `KnowledgeChunk.sourceId`
field for idempotency and `tags[]` for metadata.

The models below would improve observability, safety and performance once we
commit to a follow-up schema migration. Copy these into
`packages/db/prisma/schema.prisma` under the "AI KNOWLEDGE BASE (RAG)" block
when ready.

---

## IngestLog — audit trail for RAG ingest

Tracks every attempt to ingest a source record. Useful for:
- Debugging "why didn't this chunk appear in search?"
- Detecting ingest backpressure or repeated failures
- Computing what % of the EHR has been ingested for a given patient/doctor

```prisma
model IngestLog {
  id           String   @id @default(uuid())
  sourceType   String   // "CONSULTATION" | "LAB_RESULT" | "PRESCRIPTION" | "EHR_DOCUMENT"
  sourceId     String   // id in the originating table
  patientId    String?  // denormalised for fast patient-scoped lookups
  doctorId     String?  // denormalised for fast doctor-scoped lookups
  chunkCount   Int      @default(0)
  status       String   // "SUCCESS" | "SKIPPED_NORMAL" | "ERROR" | "OCR_PENDING"
  errorMessage String?
  createdAt    DateTime @default(now())

  @@index([sourceType, sourceId])
  @@index([patientId])
  @@index([doctorId])
  @@index([status])
  @@map("ingest_logs")
}
```

## OCR dependency note

`ingestEhrDocument` currently stubs OCR for non-text files. Production wiring
requires ONE of:

1. **tesseract.js** (OSS, local) — fastest to ship, no external API.
   - Add dep: `npm i tesseract.js`
   - Call: `const { data: { text } } = await Tesseract.recognize(filePath, 'eng')`
   - Caveats: CPU-bound, needs a worker pool for throughput.

2. **Google Cloud Vision** (managed, India region available)
   - Add dep: `npm i @google-cloud/vision`
   - Env: `GCP_SERVICE_ACCOUNT_JSON`
   - Pros: handwriting + Indic-script support, DPDP-friendly India region.

3. **AWS Textract** (managed, if the deployment is on AWS)
   - Add dep: `npm i @aws-sdk/client-textract`
   - Best for structured medical forms (lab reports with tables).

Whichever we pick, wire it in the `extractTextFromDocument` function of
`apps/api/src/services/ai/rag-ingest.ts`.
