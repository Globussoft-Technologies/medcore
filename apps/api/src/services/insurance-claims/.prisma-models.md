# Proposed Prisma models — Insurance TPA Claims

These models are **not** added to `packages/db/prisma/schema.prisma` yet (the
task brief explicitly forbade modifying the schema). They capture the shape we
need once we are ready to run the migration.

The `apps/api/src/routes/insurance-claims.ts` route currently persists claims
to an **in-process store** (`store.ts` sibling module). When we run the
migration below, swap the store reads/writes for `prisma.insuranceClaim.*`
calls — the field names line up 1:1.

## `InsuranceClaim`

> ℹ️ The existing `InsuranceClaim` model in `schema.prisma` (line 925) is a
> lightweight row with just `status: ClaimStatus { SUBMITTED | APPROVED |
> REJECTED | SETTLED }`. We propose renaming the old row to `InsuranceClaimLegacy`
> and introducing the richer model below, or extending the existing one with
> the new columns. Pick one on migration day.

```prisma
model InsuranceClaim {
  id                String                @id @default(uuid())

  // FK to Invoice (Invoice doubles as "Bill" in MedCore).
  billId            String
  patientId         String

  // TPA routing.
  tpaProvider       TpaProvider
  providerClaimRef  String?               @unique  // null until submit succeeds
  insurerName       String                // e.g. "Star Health"
  policyNumber      String
  memberId          String?

  // Linkage to an approved pre-auth, if any.
  preAuthRequestId  String?

  // Clinical / financial.
  diagnosis         String
  icd10Codes        Json?                 // string[]
  procedureName     String?
  admissionDate     DateTime?             @db.Date
  dischargeDate     DateTime?             @db.Date
  amountClaimed     Float
  amountApproved    Float?

  // Lifecycle.
  status            NormalisedClaimStatus @default(SUBMITTED)
  deniedReason      String?
  notes             String?
  submittedAt       DateTime              @default(now())
  approvedAt        DateTime?
  settledAt         DateTime?
  cancelledAt       DateTime?
  lastSyncedAt      DateTime?             // when we last polled the TPA

  createdBy         String
  createdAt         DateTime              @default(now())
  updatedAt         DateTime              @updatedAt

  invoice           Invoice               @relation(fields: [billId], references: [id])
  patient           Patient               @relation(fields: [patientId], references: [id])
  preAuthRequest    PreAuthRequest?       @relation(fields: [preAuthRequestId], references: [id])
  documents         ClaimDocument[]
  events            ClaimStatusEvent[]

  @@index([billId])
  @@index([patientId])
  @@index([tpaProvider, status])
  @@index([providerClaimRef])
  @@map("insurance_claims_v2")
}
```

## `ClaimDocument`

One row per supporting document we upload to the TPA. We keep `fileKey` so we
can serve it back through `getSignedDownloadUrl()` without re-fetching from
the TPA.

```prisma
model ClaimDocument {
  id             String   @id @default(uuid())
  claimId        String
  type           String   // DISCHARGE_SUMMARY | INVESTIGATION_REPORT | PRESCRIPTION | BILL | ID_PROOF | CONSENT_FORM | OTHER
  fileKey        String   // storage key — uploadFile() return value
  filename       String
  contentType    String
  sizeBytes      Int
  providerDocId  String?  // returned by TPA; null if upload failed or not yet done
  uploadedBy     String
  uploadedAt     DateTime @default(now())

  claim          InsuranceClaim @relation(fields: [claimId], references: [id], onDelete: Cascade)

  @@index([claimId])
  @@map("claim_documents")
}
```

## `ClaimStatusEvent`

Append-only timeline. Every time we poll (or receive a webhook from) the TPA
we insert a row. The route's `GET /claims/:id` response returns these in
ascending order so the UI can render a vertical stepper.

```prisma
model ClaimStatusEvent {
  id         String                @id @default(uuid())
  claimId    String
  status     NormalisedClaimStatus
  timestamp  DateTime              @default(now())
  note       String?
  source     String                @default("API") // API | WEBHOOK | MANUAL
  createdBy  String?               // userId when source=MANUAL

  claim      InsuranceClaim @relation(fields: [claimId], references: [id], onDelete: Cascade)

  @@index([claimId, timestamp])
  @@map("claim_status_events")
}
```

## Migration checklist

1. Add the enums + 3 models above to `schema.prisma`.
2. Run `pnpm prisma migrate dev --name insurance_claims_v2`.
3. Swap `apps/api/src/services/insurance-claims/store.ts` reads/writes for
   `prisma.insuranceClaim.*` / `prisma.claimDocument.*` /
   `prisma.claimStatusEvent.*` calls.
4. Drop the in-memory store once verified.
5. Back-fill the old `InsuranceClaim` table into `insurance_claims_v2` with a
   one-shot script if there is production data to preserve.
