# MedCore i18n (English / Hindi)

MedCore uses a **flat dictionary** approach — no `i18next`, no `next-intl`, no
heavyweight runtime. Strings live in [`src/lib/i18n.ts`](./i18n.ts) as two
parallel `Record<string, string>` objects (`en` and `hi`) keyed by the same
namespaced strings.

## Why flat instead of a library

- **Zero runtime cost** — single small store, no async loaders.
- **Type-checked at build time** — TypeScript verifies every key passes through
  `t()` correctly.
- **Easy review** — translators see English and Hindi side by side in one file.
- **Works in Server + Client components** — `useTranslation()` is client-only,
  but Server Components can statically import the dict if needed.

This will scale to a few thousand keys. If/when the dict crosses ~5k entries
or the team needs ICU pluralization / RTL / multiple non-Latin scripts, swap to
`i18next` (the public surface — `useTranslation()` returning `{ t, lang, setLang }`
is intentionally i18next-compatible to make migration trivial).

## How to add a new string

1. Open `src/lib/i18n.ts`.
2. Add the key to **both** `en` and `hi` objects. Group it under the right
   namespace block (e.g. `dashboard.appointments.*`, `dashboard.actions.*`,
   `common.*`, `login.*`, `register.*`, `feedback.*`).

   ```ts
   // en
   "dashboard.lab.archiveBtn": "Archive Order",
   // hi
   "dashboard.lab.archiveBtn": "आदेश संग्रह करें",
   ```

3. Use it in your component:

   ```tsx
   "use client";
   import { useTranslation } from "@/lib/i18n";

   export function ArchiveButton() {
     const { t } = useTranslation();
     return <button>{t("dashboard.lab.archiveBtn")}</button>;
   }
   ```

4. Optionally pass a fallback (rendered if the key is missing):

   ```tsx
   t("dashboard.lab.archiveBtn", "Archive Order")
   ```

## Naming conventions

- `app.*` — brand-level strings (name, tagline)
- `common.*` — generic UI verbs / nouns shared across pages (Cancel, Save,
  Search, Loading, Date, etc.)
- `login.*`, `register.*`, `feedback.*` — patient-facing public pages
- `dashboard.nav.*` — sidebar / bottom nav labels
- `dashboard.actions.*` — buttons that act on resources (Reschedule, Discharge,
  Collect Payment)
- `dashboard.<module>.*` — per-module strings: `appointments`, `patients`,
  `billing`, `admissions`, `lab`, `emergency`, `queue`, `prescriptions`,
  `home`. Keep these grouped — sub-namespaces are encouraged
  (`dashboard.appointments.col.token`, `dashboard.appointments.status.booked`).

## Rules

- **Always add both languages.** A missing Hindi string falls back to English
  silently — that's a footgun for the audit. Always commit the Hindi version
  in the same change.
- **Don't translate medical units, drug names, or codes.** Leave `mg`, `°C`,
  `BP`, `ICD-10`, MRN, and trade names untranslated.
- **Don't translate identifiers shown to staff for cross-system lookups**
  (token numbers, invoice IDs, MR numbers).
- **Hard-code if the string is fully internal / debug-only.** Toast messages,
  error logs, and console output remain English. Only translate user-visible
  copy.
- **Prefer interpolation outside `t()`.** No `{{var}}` syntax — just compose:

  ```tsx
  `${t("dashboard.queue.minWait")} (${minutes} min)`
  ```

## Switching language

`<LanguageDropdown />` writes to `localStorage["medcore_lang"]` and updates
`document.documentElement.lang` so screen readers announce the correct
pronunciation. The root `<html lang>` is also seeded by an inline pre-hydration
script in `apps/web/src/app/layout.tsx` to avoid a flash of the wrong language.

## Adding a new locale (e.g. Tamil, Bengali)

1. Add to the `Lang` union in `i18n.ts`: `export type Lang = "en" | "hi" | "ta";`
2. Add a new dict const `const ta: Dict = { ... }` mirroring all keys.
3. Add it to `translations`: `{ en, hi, ta }`.
4. Update `LanguageDropdown.tsx` to expose the new `<option>`.
5. Update the inline script in `apps/web/src/app/layout.tsx` to recognise the
   new code.

## Audit / coverage

To find untranslated strings, grep for hard-coded English in JSX:

```bash
# rough — finds bare-string children that look like sentences
rg '>[A-Z][a-z]+(?: [a-z]+){1,5}<' apps/web/src/app/dashboard
```

When you see one in user-visible UI, namespace it under `dashboard.<module>.*`
and replace the literal with `t()`.
